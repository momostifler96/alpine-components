Alpine Components

Bibliothèque headless de composants UI pour Alpine.js 3, compatible Livewire et Laravel. Chaque composant = factory Alpine.data retournant état + méthodes. Vous possédez le HTML et le style.

Alpine.js 3.14+ Tailwind CSS 4 Livewire ready ESM + UMD 0 dépendance runtime
npm → GitHub →

Installation

npm install @momoledev/alpine-components

Avec le plugin Alpine (recommandé)

import '@momoledev/alpine-components/style'   // classes ap-* + tokens CSS
import Alpine from 'alpinejs'
import AlpineComponents from '@momoledev/alpine-components'

Alpine.plugin(AlpineComponents)
Alpine.start()

Importer uniquement certains composants

import Alpine from 'alpinejs'
import { apSelect, apForm, registerToastStore } from '@momoledev/alpine-components'

registerToastStore(Alpine)
Alpine.data('apSelect', apSelect)
Alpine.data('apForm', apForm)
Alpine.start()

Sans style (headless pur)

Omettez l'import du style et appliquez vos propres classes Tailwind.

import Alpine from 'alpinejs'
import AlpineComponents from '@momoledev/alpine-components'

Alpine.plugin(AlpineComponents)
Alpine.start()

Style — classes ap-*

Le package inclut un fichier CSS optionnel avec des classes ap-* prêtes à l'emploi, basées sur des variables CSS --ap-* entièrement personnalisables. 

import '@momoledev/alpine-components/style'

Ou dans un fichier CSS (Laravel app.css) :

@import '@momoledev/alpine-components/style';

Personnaliser les tokens CSS

Surchargez les variables --ap-* après l'import :

:root {
  --ap-primary:        #10b981;   /* couleur principale */
  --ap-primary-hover:  #059669;
  --ap-primary-light:  #ecfdf5;
  --ap-primary-ring:   #a7f3d0;
  --ap-radius:         0.5rem;    /* coins des champs */
  --ap-height:         2.25rem;   /* hauteur des champs */
}

Exemple avec classes ap-*

<form class="ap-form"
      x-data="apForm({ data: { email: '' } })"
      @submit.prevent="submit('/api/register')">

  <div class="ap-form-field">
    <label class="ap-form-label is-required">Email</label>
    <input class="ap-input" :class="{ 'is-error': hasError('email') }"
           x-model="data.email" type="email" placeholder="vous@exemple.com">
    <p class="ap-form-error" x-show="hasError('email')" x-text="getError('email')"></p>
  </div>

  <button type="submit" class="ap-form-submit" :disabled="loading">
    <span x-show="!loading">S'inscrire</span>
    <span x-show="loading">Envoi…</span>
  </button>
</form>

Via CDN

Sans npm, ajoutez directement dans votre HTML :

<!-- Style prédéfini -->
<link rel="stylesheet"
  href="https://cdn.jsdelivr.net/npm/@momoledev/alpine-components/dist/style.css">

<!-- Alpine.js -->
<script defer src="https://cdn.jsdelivr.net/npm/alpinejs@3/dist/cdn.min.js"></script>

<!-- Composants (ESM) -->
<script type="module">
  import AlpineComponents from 'https://cdn.jsdelivr.net/npm/@momoledev/alpine-components/dist/index.mjs'
  document.addEventListener('alpine:init', () => Alpine.plugin(AlpineComponents))
</script>

Laravel + Vite

1. Installer

npm install @momoledev/alpine-components

2. Importer le style dans resources/css/app.css

@import '@momoledev/alpine-components/style';

Alternative — dans resources/js/app.js :

import '@momoledev/alpine-components/style'

3. Enregistrer dans resources/js/app.js

import Alpine from 'alpinejs'
import AlpineComponents from '@momoledev/alpine-components'

Alpine.plugin(AlpineComponents)

window.Alpine = Alpine
Alpine.start()

Architecture

Modèle headless : la factory retourne état + méthodes ; vous possédez le HTML/CSS. Les champs bindables émettent input et change pour Livewire. 

src/
  components/   → apSelect, apDropdown, apInputText, apInputTags,
                   apSwitch, apSlider, apForm, apInputMask, Toast
  data/         → catalog, phoneCountries
  utils/        → formatMoney, demoCopy
  index.js      → exports npm
  main.js       → playground de démo (non publié)

Contrat d'un composant

  • export default function apXxx(config = {}) { … }
  • Valeurs par défaut via ?? (pas ||)
  • Pas d'import Alpine — les factories sont de pures fonctions
  • Composants bindables : $dispatch('input', val) + $dispatch('change', val)

Livewire

Tous les composants dispatchent input et change à chaque changement de valeur — ils sont nativement compatibles avec wire:model.live. Placez x-modelable et wire:model.live sur le même élément conteneur.

Select (simple ou multiple)

{{-- Composant Livewire --}}
<div
    x-data="apSelect(@js(['options' => $options, 'multiple' => true]))"
    x-modelable="value"
    wire:model.live="selectedTechs"
    @click.outside="close()"
    class="ap-select"
>
    <button class="ap-select-trigger" :class="{ 'is-open': open }" @click="toggleOpen()">
        <div class="ap-select-tags" x-show="multiple && hasValue">
            <template x-for="opt in selectedOptions" :key="opt.value">
                <span class="ap-select-tag">
                    <span class="ap-select-tag-label" x-text="opt.label"></span>
                    <button type="button" class="ap-select-tag-remove" @click.stop="removeTag(opt.value, $event)">✕</button>
                </span>
            </template>
        </div>
        <span class="ap-select-trigger-label" :class="{ 'is-placeholder': !hasValue }"
              x-show="!multiple || !hasValue" x-text="display || placeholder"></span>
        <svg class="ap-select-trigger-chevron" fill="none" viewBox="0 0 24 24" stroke="currentColor" stroke-width="2"><path stroke-linecap="round" stroke-linejoin="round" d="M19 9l-7 7-7-7"/></svg>
    </button>
    <div class="ap-select-panel" x-show="open" x-cloak>
        <ul class="ap-select-options">
            <template x-for="opt in filteredOptions" :key="opt.value">
                <li class="ap-select-option" :class="{ 'is-selected': isSelected(opt.value) }"
                    @click="select(opt)" x-text="opt.label"></li>
            </template>
        </ul>
    </div>
</div>

Switch

<div x-data="apSwitch()" x-modelable="value" wire:model.live="darkMode">
    <button type="button" @click="toggle()" :disabled="disabled"
        class="ap-switch" :class="{ 'is-on': value }" role="switch" :aria-checked="value">
        <span class="ap-switch-thumb"></span>
    </button>
</div>

InputTags

<div x-data="apInputTags()" x-modelable="tags" wire:model.live="skills">
    <div class="ap-tags-wrapper" @click="$refs.tagInput.focus()">
        <template x-for="(tag, i) in tags" :key="i">
            <span class="ap-tag">
                <span class="ap-tag-label" x-text="tag"></span>
                <button type="button" class="ap-tag-remove" @click.stop="removeTag(i)">✕</button>
            </span>
        </template>
        <input x-ref="tagInput" x-model="input" class="ap-tags-input"
               @keydown="onKeydown($event)" @blur="input.trim() && addTag(input)">
    </div>
</div>

apForm avec Livewire (erreurs 422)

apForm récupère automatiquement les erreurs de validation Laravel renvoyées en JSON (status 422) et les affiche via hasError() / getError().

{{-- Pré-remplir les erreurs issues d'un redirect Livewire --}}
<form class="ap-form"
      x-data="apForm({
          data: { name: '', email: '' },
          errors: @json($errors->getMessages()),
      })"
      @submit.prevent="submit('{{ route('api.members.store') }}')">

    <div class="ap-form-field">
        <label class="ap-form-label is-required">Nom</label>
        <input class="ap-input" :class="{ 'is-error': hasError('name') }"
               x-model="data.name" placeholder="Jean Dupont">
        <p class="ap-form-error" x-show="hasError('name')" x-text="getError('name')"></p>
    </div>

    <div class="ap-form-success" x-show="success">✓ Membre enregistré.</div>

    <button type="submit" class="ap-form-submit" :disabled="loading" :class="{ 'is-loading': loading }">
        <span x-show="!loading">Enregistrer</span>
        <span x-show="loading">Envoi…</span>
    </button>
</form>
Comment ça marche : chaque composant avec x-modelable observe sa propriété interne (ex. value) et dispatche input + change via $dispatch. Livewire intercepte ces événements DOM natifs — aucun code supplémentaire requis.
